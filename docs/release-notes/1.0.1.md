### What's changed

The AirPods microphone now works on Windows.

#### Fixed

- **AirPods microphone is now usable as a Windows input device.** Previously the
  AirPods appeared only as a playback device and the mic stayed silent — the tray
  read *"Mic: couldn't determine"* and the mic policy quietly did nothing. In
  **Auto-switch** (during a call) or **Call-mode**, PodBridge now actively engages
  the Hands-Free (HFP) link — through a silent, communications-category audio
  session, still with **no driver and no admin** — so Windows brings the AirPods
  microphone live and routes your call to it. Media returns to stereo A2DP when
  the call ends. ([#156](https://github.com/bhemsen/PodBridge/issues/156))

#### Added

- **Honest "microphone unavailable" notice.** When a mic mode asks for the AirPods
  microphone but Windows cannot expose it — for example when the Hands-Free /
  Headset profile is not enabled for the device — PodBridge now says so plainly
  (*"AirPods mic unavailable — enable Hands-Free/Headset for the AirPods in Windows
  Bluetooth settings, or re-pair"*) instead of failing silently.

#### Changed

- Documentation: added a CONTRIBUTING guide and issue templates, a README
  screenshot and a feature-comparison table, and pointed the download link at the
  latest release.
