### What's changed

The optional advanced tier (noise-control switching) now actually works on real
hardware, and its driver is a downloadable release.

#### Fixed

- **Advanced-tier driver now loads and binds on real AirPods.** The Tier-2 KMDF
  L2CAP driver bound to an invented Bluetooth service GUID the AirPods never
  advertise, so Windows never created a device node and the driver never loaded —
  the `Noise control` submenu stayed disabled on every machine. It now binds to
  the Apple AAP service UUID the AirPods actually advertise, so noise-control
  switching (Off / Noise Cancellation / Transparency / Adaptive) works with
  echo-confirmation on hardware (validated on AirPods Pro 2).
  ([#159](https://github.com/bhemsen/PodBridge/issues/159))

#### Added

- **The advanced-tier driver ships as its own download.** It is published as a
  separate, test-signed `driver-v*` GitHub release (never bundled in the app),
  with a checksum and a build-provenance attestation. The in-app *Enable advanced
  tier…* flow and the [advanced-tier guide](https://github.com/bhemsen/PodBridge/blob/main/docs/user/advanced-tier.md)
  now point at that download instead of requiring you to build the driver
  yourself — local building stays available as an alternative.
  ([#162](https://github.com/bhemsen/PodBridge/pull/162))

Unchanged honesty: the driver is **test-signed, not Microsoft-signed** — loading
it still requires enabling test-signing mode and trusting the included test
certificate, both opt-in and reversible. Every default (Tier-1) feature keeps
working with no driver and no admin.
